# Описание программы EduSphere

**Обозначение:** EduSphere v1.0  
**Стандарт оформления:** ГОСТ 19.402-78, ГОСТ 19.404-79

---

## 1. Общие сведения

### 1.1. Назначение

EduSphere — глобальная образовательная платформа с адаптивным обучением, предлагающая индивидуальные учебные планы и виртуальные классы с дополненной реальностью (AR).

### 1.2. Класс системы

| № | Класс | Обоснование |
|---|-------|-------------|
| 1 | Средний уровень | Представление образовательных услуг |
| 2 | Образование | Автоматизация обучения, оценки, взаимодействия |
| 3 | Частично интегрированная | Микросервисы взаимодействуют через API, единый UI |
| 4 | Клиент-серверная | Данные на сервере, доступ через веб и мобильные приложения |
| 5 | Графический интерфейс | Визуальные элементы для упрощения обучения |
| 6 | Обработка всех типов данных | Текст, числа, графика, звук, видео, AR-модели |
| 7 | Информационно-справочная ИС | Предоставление образовательной информации |

### 1.3. Условия функционирования

- Сервер: Linux/Windows, Java 21, PostgreSQL 16, Docker
- Клиент веб: Windows 10+, macOS Sequoia 15+, современный браузер
- Клиент мобильный: Android 14+, iOS 18+ с поддержкой AR

### 2.1. Модули системы

| Модуль | Назначение |
|--------|-----------|
| **Ядро адаптивного обучения** | Анализ действий ученика, динамическая подстройка сложности и типа контента |

## 3. Используемые алгоритмы

### 3.1. Генерация учебного плана (ИИ)

1. Получение целей, интересов и предметов ученика
2. Формирование модулей по предметам с оценкой часов
3. Добавление элективных модулей по интересам
4. Расчёт confidence score (базовый 0.75 + бонусы за полноту данных)
5. Сохранение плана со статусом `AI_GENERATED`
6. Утверждение/корректировка преподавателем → `TEACHER_APPROVED`

### 3.2. Адаптивные рекомендации

1. Агрегация learning events по ученику
2. Расчёт среднего балла по предметам
3. Определение сложности: score ≥ 0.85 → ADVANCED, ≥ 0.65 → INTERMEDIATE, иначе BEGINNER
4. Выбор типа контента: низкий score → VIDEO, высокий → TEXT/IMAGE
5. Confidence: 0.85 + 0.005 × min(events, 20), max 0.96

### 3.3. Шифрование данных

- Алгоритм: AES-256-GCM
- Соответствие: ФЗ-152 (РФ), GDPR (ЕС), FERPA (США)

---

## 4. Взаимодействие компонентов

1. Клиент аутентифицируется через Auth Service → JWT
2. API Gateway маршрутизирует запросы к микросервисам
3. Curriculum Service генерирует план → сохраняет в PostgreSQL
4. Adaptive Service записывает события → формирует рекомендации
5. Analytics Service агрегирует данные из Grading и Adaptive
6. AR Session Service управляет сессиями, Mobile App рендерит 3D-модели
